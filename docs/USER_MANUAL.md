# ItemClinic 사용자 매뉴얼

시험 결과 파일을 올리면 문항분석 → 불량문항 진단 → 개선 문항 생성까지
자동으로 처리해 주는 도구입니다. 이 문서는 화면을 따라 하면 되는 사용법 안내입니다.

---

## 1. 시작하기

### 방법 ① 웹에서 바로 사용 (권장 — 설치 불필요)

**https://jp5678.github.io/ItemClinic/**

<img src="qr-itemclinic.png" alt="ItemClinic 접속 QR 코드" width="180">

휴대폰·태블릿·다른 컴퓨터 어디서든 QR 코드를 스캔하거나 주소로 접속하면
바로 사용할 수 있습니다. **모든 계산은 접속한 브라우저 안에서만 수행되며,
시험 파일이 어떤 서버로도 전송되지 않습니다.**

### 방법 ② 내 컴퓨터에서 실행 (문항은행 누적 기능 포함)

터미널을 열고:

```bash
cd ~/Desktop/iMac_DATA/ItemClinic
python3 -m itemclinic serve
```

브라우저에서 **http://127.0.0.1:8765** 로 접속. 종료는 터미널에서 `Ctrl+C`.

필요한 것: Python 3.10 이상 · 엑셀 분석 시 `pip3 install openpyxl` ·
API 자동 생성 시 `pip3 install anthropic` (모두 선택)

> 두 방법의 화면과 분석 결과는 동일합니다. 차이는 **문항은행 자동 누적과
> CLI 검색(7장)이 방법 ②에만 있다는 점**입니다.

## 2. 업로드 화면 입력 항목

| 항목 | 설명 |
|---|---|
| **학년도 / 학기** | 드롭다운 선택. 시험 이름 앞에 "2026-1" 형식으로 자동으로 붙습니다. |
| **시험 유형** | 중간·기말·퀴즈(규준참조) / 진단·형성평가(준거참조). **유형에 따라 판정 기준이 달라집니다** — 예: 형성평가는 정답률 95%가 정상. |
| **시험 이름** | 과목명 등 (예: "성인간호학 중간고사"). 비워 두면 문항 분석표의 응시과목이 자동 사용됩니다. |
| **과목명** | 선택. 개선 문항 생성 프롬프트에 쓰입니다. |
| **응답 파일** | CSV 또는 엑셀(.xlsx). 아래 3. 지원 형식 참고. |
| **상단 제외 행 수** | 파일 위쪽 제목 영역 행 수. 학교 출력물은 기본값 7 그대로 두면 됩니다. 1행부터 데이터면 0. |
| **문항 원문 JSON** | 선택. 문항 텍스트를 주면 개선안 품질이 좋아집니다. |
| **Claude API 키** | 선택. 입력하면 개선 문항이 리포트에 바로 자동 생성됩니다. 키는 이 브라우저에만 저장됩니다. |

## 3. 지원 파일 형식

### ① 학교 '문항 분석표' (자동 인식 — 가장 간단)

학교 시스템에서 내려받은 문항 분석표(문항별 정답·최다오답·답지반응률·정답률·
변별도 표)를 **그대로** 올리면 됩니다. 아무 설정도 필요 없습니다.
상단의 응시학과·응시과목·시험일자·응시인원·평균·표준편차를 자동으로 읽어
리포트에 반영합니다.

> 이 형식은 학생별 원 응답이 아니므로 KR-20 신뢰도와 문항특성곡선(ICC)은
> 산출되지 않습니다 (리포트에 자동 표기).

### ② 학생별 응답 파일 (KR-20·ICC까지 산출)

첫 열 = 학생 ID, 이후 열 = 문항. 두 가지 방식:

```csv
student,Q1,Q2,Q3        ← 헤더
KEY,A,B,C               ← 정답 키 행 (원 응답 방식, 오답지 분석 가능)
S001,A,B,D
S002,A,C,C
```

또는 KEY 행 없이 모든 셀이 0/1인 기채점 행렬. 빈 셀(무응답)은 오답 처리됩니다.

## 4. 리포트 읽는 법

### 상단 카드

- **KR-20 신뢰도**: 0.7 이상 수용, 0.8 이상 양호 (학생별 응답 파일일 때만)
- **주의·불량 문항 수**: 아래 상세에서 원인 확인

### 문항 판정 기준 (규준참조 기본)

| 판정 | 조건 | 조치 |
|---|---|---|
| **불량** (빨강) | 변별도 음수 / 0.10 미만 / 정답보다 많이 선택된 오답 | 정답 확인 → 폐기·전면 수정 |
| **주의** (주황) | 변별도 0.20 미만, 정답률 0.20 미만·0.90 초과, 삭제 시 KR-20 상승 | 오답지 점검·부분 수정 |
| **참고** (파랑) | 선택률 5% 미만 오답지 | 다음 개정 때 교체 |

**변별도가 음수인 문항이 가장 심각합니다** — 성적이 좋은 학생일수록 오히려
틀린다는 뜻으로, 정답 오류·중의적 표현·상위권을 속이는 오답이 의심됩니다.

### 문항별 상세

- **ICC 곡선** (학생별 응답일 때): 우상향이 정상, 평평하거나 우하향이면 불량
- **답지반응률 표**: ✓ 표시가 정답. 상위 27%가 정답보다 특정 오답을 많이
  골랐다면 그 오답지를 의심하세요.

## 5. 리포트 저장 (PDF · DOCX · JSON)

웹 버전 리포트 **우측 상단의 "리포트 저장" 버튼**으로 전체 리포트
(검사 개요 + 문항 통계 요약 + 문항별 상세 진단 + Claude 개선안 + 개선 프롬프트)를
파일로 내려받을 수 있습니다.

| 버튼 | 형식 | 용도 |
|---|---|---|
| **PDF** | 인쇄 화면 → "PDF로 저장" 선택 | 보관·제출용. ICC 차트와 색상 그대로 저장 |
| **DOCX** | Word 문서 | 학과 보고서에 붙여넣거나 편집할 때 |
| **JSON** | 진단 데이터 원자료 | 학기별 누적 보관, 엑셀·Claude 재분석용 |

- PDF 버튼을 누르면 브라우저 인쇄 창이 열립니다 — 대상에서 **"PDF로 저장"**을
  선택하세요. 입력 패널은 자동으로 제외되고 리포트만 저장됩니다.
- 파일 이름은 시험 이름으로 자동 지정됩니다 (예:
  `2026-1-성인간호학-중간고사_문항분석리포트.docx`).
- 로컬 Python 버전은 분석할 때마다 `report.html`·`diagnosis.json`·
  `improve_prompt.md`가 폴더에 자동 저장됩니다.

## 6. 개선 문항 받기

### 방법 A — API 키 없이 (기본)

1. 리포트의 **개선 프롬프트** 제목 옆 복사 아이콘 클릭
2. Claude(claude.ai)나 Claude Code에 붙여넣기
3. 원인 해석 + 수정안 + 대체 문항 + 오답지 설계 근거를 받습니다

### 방법 B — API 키로 자동 생성

업로드할 때 API 키를 입력하면 리포트에 **Claude 개선안** 섹션이 자동으로
포함되고 `improved_items.md`로도 저장됩니다.
키는 [console.anthropic.com](https://console.anthropic.com)에서 발급(유료 종량제),
**이 브라우저에만 저장**되며 서버·파일에는 남지 않습니다.
키 칸을 비우고 제출하면 저장된 키가 삭제됩니다.

## 7. 문항은행

분석할 때마다 문항별 통계가 `item_bank.json`에 자동 누적됩니다.
학기가 쌓이면 재출제 후보를 조건 검색할 수 있습니다:

```bash
# 불량 이력 문항 모아 보기
python3 -m itemclinic bank-search --severity CRITICAL
# 재출제 후보: 난이도 0.4~0.7 & 변별도 0.3 이상
python3 -m itemclinic bank-search --min-p 0.4 --max-p 0.7 --min-r 0.3
```

## 8. 자주 묻는 질문

**Q. "헤더와 데이터 행을 찾을 수 없습니다" 오류가 나요.**
상단 제외 행 수가 파일과 맞지 않는 경우입니다. 데이터 헤더가 8행부터면 7,
1행부터면 0으로 조정하세요. (문항 분석표는 이 설정과 무관하게 자동 인식됩니다.)

**Q. 값 'B'를 해석할 수 없다는 오류가 나요.**
원 응답(A~E) 파일인데 KEY 행이 없는 경우입니다. 헤더 바로 아래에
`KEY,정답,정답,...` 행을 추가하세요.

**Q. 구형 .xls 파일이 안 열려요.**
엑셀에서 "다른 이름으로 저장 → Excel 통합 문서(.xlsx)"로 변환 후 올리세요.

**Q. 결과 파일은 어디에 저장되나요?**
웹 분석은 `web_analyses/시험이름_날짜시각/` 폴더에 report.html,
diagnosis.json, improve_prompt.md(, improved_items.md)가 저장됩니다.

**Q. 학생 데이터가 외부로 전송되나요?**
아니요. 웹 버전은 접속한 브라우저 안에서만 계산하고, 로컬 버전 서버는 이
컴퓨터(127.0.0.1) 안에서만 동작합니다. 단, API 키를 입력한 경우 개선
프롬프트(문항 통계 요약)가 Anthropic API로 전송됩니다 — 학생 개인 식별
정보는 프롬프트에 포함되지 않습니다.

**Q. 웹 버전 결과는 어떻게 보관하나요?**
리포트 우측 상단의 "리포트 저장" 버튼으로 PDF·DOCX·JSON 파일로 내려받아
보관하세요 (5장 참고).
