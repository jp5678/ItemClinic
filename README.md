# ItemClinic — 문항은행 + 자동 문항분석 엔진

객관식 시험 결과 CSV 하나로 **KR-20 신뢰도 → 난이도·변별도 → 문항특성곡선(ICC) →
오답지 매력도 → 불량문항 진단 → 개선 문항 생성 프롬프트**까지 한 번에 산출하는
파이프라인입니다. 진단평가·형성평가·중간고사·기말고사·퀴즈 등 **시험 유형별로
진단 기준이 자동 조정**됩니다. 외부 라이브러리 없이 Python 표준 라이브러리만 사용합니다.

## 가장 빠른 시작 — 웹에서 바로 사용 (설치 불필요)

**https://jp5678.github.io/ItemClinic/** 에 접속해 파일을 올리면 끝입니다.
모든 계산은 접속한 브라우저 안에서만 수행되며 시험 데이터가 서버로 전송되지
않습니다. (엑셀 읽기 라이브러리만 CDN에서 로드)

## 로컬 웹 UI — Python 서버

```bash
cd ItemClinic
python3 -m itemclinic serve            # http://127.0.0.1:8765
```

브라우저에서 시험 유형 선택 → CSV 업로드 → 분석 시작을 누르면 리포트가 바로 열립니다.
결과 파일은 `web_analyses/` 아래에 저장되고 문항은행에 자동 누적됩니다.
화면 하단 문구는 `--footer "청암대학교 간호학과 · ○○○ 교수"` 처럼 바꿀 수 있습니다.

## 명령행 사용

```bash
python3 -m itemclinic analyze 시험결과.xlsx --exam-id 2026-1-중간 \
    --exam-type 중간고사 --subject "과목명" --skip-rows 7
```

`--exam-type`: 일반(기본) · 중간고사 · 기말고사 · 퀴즈 · 진단평가 · 형성평가

산출물 (`<exam-id>_analysis/` 폴더):

| 파일 | 내용 |
|---|---|
| `report.html` | 한국어 분석 리포트 (ICC 차트, 오답지 분포표, 문항별 진단 내장) |
| `diagnosis.json` | 전체 통계·플래그 구조화 데이터 |
| `improve_prompt.md` | 불량문항 개선 요청 프롬프트 (Claude에 붙여넣기용) |
| `item_bank.json` | 문항은행 — 실행할 때마다 문항 통계 이력이 누적 |

## 입력 파일 형식 (CSV / 엑셀)

`.csv`와 엑셀 `.xlsx`(첫 번째 시트)를 모두 지원합니다.
학교 시스템 출력물처럼 **상단에 제목 영역이 있으면 '상단 제외 행 수'로 건너뜁니다**
— 웹 UI 기본값은 7 (1~7행 제외, 데이터 헤더가 8행부터 시작),
CLI는 `--skip-rows 7`. 파일 1행부터 데이터면 0으로 지정하세요.
구형 `.xls`는 엑셀에서 `.xlsx`로 다른 이름 저장 후 사용합니다.
엑셀 응답이 숫자(1~5)여도 자동으로 정수 문자열로 정규화되어 정답 키와 비교됩니다.

### 학교 '문항 분석표'(요약 형식) 자동 인식

학교 시스템이 출력하는 **문항 분석표**(문항별 정답·최다오답·답지반응률·정답률·
변별도·난이도 표)를 올리면 자동 감지되어 별도 설정 없이 분석됩니다:

- 상단 텍스트 영역(응시학과·응시과목·시험일자·응시인원·전체평균·표준편차·정답률)을
  읽어 시험 이름과 리포트 카드에 자동 반영 — 시험 이름을 비워 두면 응시과목이 쓰입니다.
- 제공된 정답률·변별도로 불량문항 진단, 답지반응률로 오답지 진단
  (정답보다 많이 선택된 오답 → 정답 오류 의심 '불량', 선택률 5% 미만 오답 → 참고 표시).
- 학생별 원 응답이 아니므로 KR-20·ICC는 산출하지 않음을 리포트에 명시합니다.
- 답지반응률 단위(% 또는 비율)는 열 단위로 자동 판별합니다.

## 데이터 배치 형식 (학생별 응답 파일)

**형식 1 — 원 응답 + 정답 키** (오답지 분석까지 가능, 권장):

```csv
student,Q1,Q2,Q3
KEY,A,B,C
S001,A,B,D
S002,A,C,C
```

**형식 2 — 기채점 0/1 행렬** (KEY 행 없이 모든 셀이 0/1):

```csv
student,Q1,Q2,Q3
S001,1,1,0
S002,1,0,1
```

빈 셀(무응답)은 오답 처리. 첫 열은 학생 ID, KEY 행 ID는 `KEY`/`정답`/`answer` 인식.

## 문항 원문 연결 (개선안 품질 향상)

문항 텍스트를 JSON으로 주면 진단·개선 프롬프트에 원문이 포함됩니다
(형식 예시: [examples/items_sample.json](examples/items_sample.json)):

```bash
python3 -m itemclinic analyze 결과.csv --items 문항원문.json
```

## 개선 문항 자동 생성

두 가지 방식을 모두 지원합니다.

**API 키 없이 (기본)** — 분석 시 `improve_prompt.md`가 생성되고 리포트의 복사
버튼으로 복사해 Claude 앱에 붙여넣거나, Claude Code에서
**"improve_prompt.md 읽고 개선안 작성해줘"** 라고 요청하면 됩니다.

**API 키로 자동 생성 (선택)** — 웹 업로드 화면의 'Claude API 키' 칸에 키를
입력하면 분석 직후 원인 해석·대체 문항이 리포트의 'Claude 개선안' 섹션과
`improved_items.md`에 자동 포함됩니다.
- 키는 **브라우저(localStorage)에만 저장**되어 다음 방문 때 자동으로 채워지며,
  서버·결과 파일·로그에는 남지 않습니다. 칸을 비우고 제출하면 저장된 키도 삭제됩니다.
- `anthropic` 패키지 필요 (`pip3 install anthropic`), 호출 실패 시에도 분석
  리포트는 정상 생성되고 프롬프트 복사 방식으로 안내됩니다.
- CLI는 `ANTHROPIC_API_KEY` 환경변수를 사용하며 `--no-llm`으로 생략 가능합니다.

## 문항은행 검색

```bash
# 불량 판정 이력 문항
python3 -m itemclinic bank-search --severity CRITICAL
# 재출제 후보: 난이도 0.4~0.7, 변별도 0.3 이상
python3 -m itemclinic bank-search --min-p 0.4 --max-p 0.7 --min-r 0.3
```

## 시험 유형별 진단 기준

| 유형 | 성격 | 기준 조정 |
|---|---|---|
| 일반·중간고사·기말고사·퀴즈 | 규준참조 (학생 간 변별) | 아래 통용 기준 그대로 |
| 진단평가 | 준거참조 (선수학습 결손 확인) | 낮은 정답률 허용 (결손의 증거), 변별도 기준 완화 |
| 형성평가 | 준거참조 (완전학습 확인) | 높은 정답률 허용 (도달의 증거), 변별도 기준 완화 |

준거참조 유형에서는 KR-20 해석 문구도 달라집니다(점수가 비슷하면 낮게 나오는 것이 정상).
**음의 변별도는 어떤 유형에서도 항상 '불량'입니다** — 문항 자체 결함의 신호이기 때문입니다.

## 진단 기준 (고전검사이론 통용 기준)

| 지표 | 기준 | 판정 |
|---|---|---|
| 변별도 r_pb (수정 양류상관) | < 0 | 불량 — 폐기/전면 수정 |
| | < 0.10 | 불량 |
| | < 0.20 | 주의 |
| 난이도 p | < 0.20 | 너무 어려움 (주의) |
| | > 0.90 | 너무 쉬움 (주의) |
| 삭제 시 KR-20 | 상승 | 신뢰도 저해 문항 (주의) |

변별도는 해당 문항을 제외한 총점과의 상관(수정 양류상관)을 사용해 과대추정을 방지하고,
전통적 상하위 27% D지수도 함께 보고합니다.

## 데모

```bash
python3 examples/make_sample.py          # 제목 7행 + N=236, 25문항 시뮬레이션 생성

```

실제 중간고사 분석 상황을 재현합니다.

## 개발

```bash
python3 -m pytest tests/ --cov=itemclinic   # 57 tests, 커버리지 88%
```

구조: `loader`(CSV 검증/채점) → `stats`(KR-20·난이도·변별도) + `icc` + `distractors`
→ `profiles`(시험 유형별 기준) → `diagnose`(플래그 규칙) → `report`(HTML) /
`improve`(Claude 연동) / `bank`(이력 누적) / `server`+`webui`(업로드 UI)
